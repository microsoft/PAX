## Overview

Version 2.0.0 is a major reliability and accuracy release. It adds the Microsoft Graph Copilot Usage Report API for efficient single-call user enumeration, introduces deterministic Interaction API-based license detection across all Entra output paths, and delivers comprehensive 429 throttling protection for every Graph API call path in the script. Unlicensed users are now cleanly separated from errors in logs and output, and the default concurrency has been tuned down for safer operation across a wider range of tenant sizes.

### Key Features

**1. Copilot Usage Report API Integration**
- User enumeration now uses the `getMicrosoft365CopilotUsageUserDetail` Graph API endpoint, retrieving all Copilot-associated users in a single paginated call instead of per-user SKU lookups.
- Automatic v1.0 → beta endpoint fallback ensures compatibility across tenant configurations.
- Entra ID + SKU dictionary remains as a fallback when the Reports API is unavailable.
- Requires the new `Reports.Read.All` application permission.

**2. Deterministic License Detection via Interaction API**
- The `hasLicense` column in all Entra output is now driven by Interaction API results: HTTP 200 confirms `True` (licensed), HTTP 403 confirms `False` (unlicensed).
- The SKU dictionary provides an initial best-effort value, which is then overwritten by the Interaction API's deterministic result.
- `-IncludeUserInfo` mode patches `hasLicense` from interaction query results, then probes any remaining unqueried Entra users via a new `Test-CopilotLicenseProbe` function.
- `-OnlyUserInfo` mode probes every user with a minimal `$top=1` Interaction API call before export.

**3. Comprehensive 429 Throttle Protection**
- Added retry logic with exponential backoff and `Retry-After` header support to all four previously unprotected Graph API call paths: license validation, parallel batch POST, pagination GETs, and sequential processing pagination.
- Parallel batches use an inline retry scriptblock (required because PowerShell runspace isolation prevents access to the script-scoped retry wrapper from `-Parallel` blocks).
- Inter-page pacing (200ms delay between pagination requests) and inter-batch pacing prevent rapid-fire request patterns that trigger the `teamsgraph.api_global_read_fallback` throttle category.

**4. Unlicensed User Classification**
- HTTP 403 "does not have a valid Copilot license" responses are now classified as `Unlicensed` status instead of `Error` in the UserStatus CSV, collection summary, progress logs, and metrics JSON.
- Dedicated `$unlicensedCount` counter prevents unlicensed users from inflating error metrics.
- Post-collection patching ensures `hasLicense=False` in Entra output for all 403-confirmed users.

**5. Reduced Default Concurrency**
- `ParallelBatchThrottle` default reduced from 25 to 5 concurrent batches for safer API behavior. Maximum remains 25 for tenants that can support higher throughput.

## Key Improvements

| Area | What Changed | Why It Matters |
|------|-------------|----------------|
| User Enumeration | Reports API provides single-call user discovery instead of N per-user SKU lookups. | Dramatically faster user enumeration for large tenants. |
| License Accuracy | Interaction API is the deterministic authority for `hasLicense`; SKU dictionary is fallback only. | Eliminates false positives/negatives from missing or new SKU GUIDs. |
| Throttle Resilience | All 4 unprotected Graph API call paths now have retry logic with exponential backoff and `Retry-After` support. | Prevents data loss and partial exports under API throttling. |
| Unlicensed Visibility | Unlicensed users separated from errors in logs, CSV status, and metrics JSON. | Clean operational signal — errors mean actual failures, not license denials. |
| Default Concurrency | `ParallelBatchThrottle` default 25 → 5. | Safer default that works across a wider range of tenant sizes and throttle limits. |
| Pagination Pacing | 200ms inter-page delay added to both parallel and sequential pagination loops. | Prevents rapid-fire request bursts that trigger throttling. |
| Permission Change | `Reports.Read.All` required; `Organization.Read.All` removed. | `Organization.Read.All` was previously listed but never actually required by the script. |
| Filter Support Testing | `Test-FilterSupport` now tries additional users when the first returns 404. | Reduces false negatives in server-side filter detection. |

## Bug Fixes

| Issue | Root Cause | Fix |
|-------|-----------|-----|
| 429 errors on license validation | License validation loop called `Invoke-MgGraphRequest` directly with no retry logic. | Replaced with `Invoke-GraphRequestWithRetry`. |
| 429 errors in parallel batch processing | Parallel batch POST and pagination GETs had no 429 handling; PowerShell runspace isolation prevented use of script-scoped retry wrapper. | Added inline retry scriptblock inside `-Parallel` block with exponential backoff. |
| 429 errors in sequential pagination | Sequential fallback path had no retry logic on pagination calls. | Replaced with `Invoke-GraphRequestWithRetry`. |
| Unlicensed users counted as errors | HTTP 403 license denials were classified as `Error` in UserStatus CSV and incremented `$errorCount`. | New `Unlicensed` status classification with dedicated counter. |
| Aggressive default concurrency | Default of 25 concurrent batches easily triggered throttling on many tenants. | Reduced default to 5. |
| No pagination pacing | Next page requested immediately after each response, creating rapid-fire request patterns. | Added 200ms delay between pagination requests. |
| Filter test false negatives | `Test-FilterSupport` only tried the first user; a 404 (no data) was indistinguishable from "filter not supported." | Now tries additional users from the target list on 404. |

## Why This Release Matters

| Question | What v2.0.0 Delivers |
|----------|----------------------|
| Do I have to change existing automation scripts? | Only if you hard-coded `Organization.Read.All` in app registration. Replace it with `Reports.Read.All` and grant admin consent. All script parameters and output formats remain backward compatible. |
| Will my exports still work during API throttling? | Yes. Every Graph API call path now has retry logic with exponential backoff and `Retry-After` header support. The default concurrency is also reduced to prevent triggering throttles in the first place. |
| How accurate is `hasLicense` now? | Deterministic. The script uses actual Interaction API responses (200 = licensed, 403 = unlicensed) rather than relying solely on SKU GUID matching. |
| What happened to unlicensed users in my error counts? | They are now classified as `Unlicensed` (not `Error`) in the UserStatus CSV, collection summary, and metrics JSON. Your error counts will be lower and more accurate. |
| Does `-OnlyUserInfo` mode detect licenses without pulling interaction data? | Yes. A new `Test-CopilotLicenseProbe` function sends a minimal `$top=1` probe to confirm license status without retrieving interaction content. |

- **Operations Teams:** Dramatically improved reliability under API throttling — retries, pacing, and conservative defaults prevent data loss on large exports.
- **License Administrators:** Deterministic `hasLicense` values eliminate ambiguity from SKU dictionary gaps or new license SKU releases.
- **Compliance & Reporting Teams:** Clean separation of unlicensed users from errors gives accurate operational metrics.
