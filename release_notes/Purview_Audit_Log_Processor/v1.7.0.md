## Key Features in v1.7.0

### PromptFilter - Message-Level Filtering
Filter Copilot messages by prompt/response type using the new `-PromptFilter` parameter:
- **`-PromptFilter Prompt`**: Extract only user prompts (Message_isPrompt = True)
- **`-PromptFilter Response`**: Extract only Copilot responses (Message_isPrompt = False)
- **`-PromptFilter Both`**: Include both prompts and responses (excludes null values)
- **`-PromptFilter Null`**: Debug mode for messages with missing isPrompt values

Two-stage filtering architecture ensures optimal performance and data quality.

### ExcludeAgents - Inverse Agent Filtering
New `-ExcludeAgents` switch filters out agent-related interactions:
- Inverse of `-AgentsOnly` for analyzing non-agent Copilot usage
- Works in both live query and replay modes
- Detailed metrics (retention rate, processing time)

### Bug Fixes
- Fixed blank `Message_isPrompt` values in explosion output
- Stage 2 filtering now properly applies during message explosion
- Improved progress bar accuracy for filtering operations

### Documentation Enhancements
- New dedicated section for Prompt and Response Filtering
- Comprehensive examples for combining filters (AgentsOnly + PromptFilter, ExcludeAgents + PromptFilter)
- Clarified two-stage filtering behavior (Stage 1: record-level, Stage 2: message-level)
- Updated parameter reference with detailed use cases
