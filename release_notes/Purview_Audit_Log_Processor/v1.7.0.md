## Overview

Version 1.7.0 delivers critical bug fixes and enhanced filtering capabilities for the Portable Audit eXporter (PAX). This release resolves PowerShell type ambiguity issues that prevented multiple filters from working together, introduces powerful new conversation filtering features, and aligns parameter naming with Microsoft's official conventions.

### Key Highlights

**Critical Bug Fixes**
- Fixed PowerShell type ambiguity errors that caused script failures when combining multiple filter switches (UserIds + AgentsOnly)
- Resolved variable name collisions between progress tracking and elapsed time calculations in all filtering sections
- Added explicit [int] type casts throughout filtering logic to prevent "ambiguous overloads" errors
- Fixed blank Message_isPrompt values appearing in output data

**Enhanced Filtering Capabilities**
- **PromptFilter Feature**: New three-mode filtering system for conversation analysis
  - `Both`: Include only conversation turns with explicit prompt/response indicators
  - `PromptsOnly`: Isolate user prompts for input analysis
  - `ResponsesOnly`: Isolate Copilot responses for output analysis
- **ExcludeAgents Switch**: Filter out agent-based interactions for cleaner datasets
- Improved Stage 1 and Stage 2 filtering logic for more accurate conversation turn isolation

**Microsoft Convention Alignment**
- Renamed `-UserId` parameter to `-UserIds` (matches Search-UnifiedAuditLog)
- Renamed `-GroupName` parameter to `-GroupNames` (supports multiple values)
- Updated all documentation and examples to reflect plural naming
- Enhanced filter display to show UserIds and GroupNames in summary section

**Reporting Improvements**
- Explosion metrics now clearly differentiate between:
  - Multi-row records (conversation threads with multiple turns)
  - Single-row records (standalone interactions)
  - Total records processed
- Better visibility into data transformation during array explosion
