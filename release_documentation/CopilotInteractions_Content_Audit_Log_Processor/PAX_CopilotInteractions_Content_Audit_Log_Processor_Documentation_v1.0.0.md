## Overview

<details open>
<summary>What It Does</summary>

The **Portable Audit eXporter (PAX) - CopilotInteractions Content Audit Log Processor** is an enterprise-grade PowerShell script that extracts Microsoft 365 Copilot interaction content directly from Microsoft Graph API. It retrieves prompts, responses, and metadata from all Copilot applications and exports them to analysis-ready CSV or Excel files.

**Core Capabilities:**

- Retrieves Copilot interaction content via **Microsoft Graph `aiInteraction` endpoint** (Copilot Interaction Export API)
  - MS Learn: [aiInteraction resource type](https://learn.microsoft.com/en-us/graph/api/resources/aiinteraction)
- Direct API access to interaction history (no mailbox searches or complex export workflows)
- Exports to structured CSV or Excel (.xlsx) with comprehensive interaction metadata
- Supports all Microsoft 365 Copilot applications: BizChat (Copilot Chat/Microsoft 365 Copilot), Teams, Outlook, Word, Excel, PowerPoint, OneNote, Loop, SharePoint, Whiteboard, Planner, Designer, Forms, Stream
- **JSON Batch Processing:** Processes up to 20 users per batch request with configurable concurrent batches (default: 25 concurrent)
- **Watermark Pattern:** Incremental processing with timestamps for massive time savings on subsequent runs
- **Entra ID Enrichment:** Optional user directory and M365 Copilot (MAC) licensing information via `-IncludeUserInfo`
- **Statistics Aggregations:** Optional stats by user, app, and date via `-IncludeStats`
- Provides detailed logging of all operations, API calls, and performance metrics
- Automatically handles module installation (Microsoft.Graph.Authentication, ImportExcel)

**Execution Modes:**

1. **Standard Mode** - Extract Copilot interaction content for all or specified users
2. **Filtered Mode** - Filter by users (`-UserPrincipalNames`), applications (`-CopilotApps`), or dates (`-StartDate`/`-EndDate`/`-DaysBack`)
3. **Incremental Mode** (`-UseWatermark`) - Only retrieve new interactions since last watermark timestamp per user
4. **User-Only Mode** (`-OnlyUserInfo`) - Export only Entra ID user directory and M365 Copilot licensing without interaction retrieval
5. **Statistics Mode** (`-IncludeStats`) - Include aggregated statistics by user, application, and date
6. **Excel Export Mode** (`-ExportWorkbook`) - Export to Excel workbook with optional multi-tab layout for stats

   - Navigate to Azure Portal > Microsoft Entra ID > App registrations > New registration

   - Redirect URI: Leave blank (not needed for client credentials flow)
   - Click "Register"

2. **Configure API Permissions (Application Permissions - NOT Delegated)**

   - API: Microsoft Graph
   - Permissions: 
     - **AiEnterpriseInteraction.Read.All** (Application)
     - **User.Read.All** (Application)
     - **Organization.Read.All** (Application)
   - Admin consent: **REQUIRED** for all three (click "Grant admin consent for [tenant]")
   
   📖 [How to Add API Permissions](https://learn.microsoft.com/entra/identity-platform/quickstart-configure-app-access-web-apis#application-permission-to-microsoft-graph)

3. **Create Client Secret**
   - Go to "Certificates & secrets" > "Client secrets" > "New client secret"
   - Description: "PAX Script Access" (or your preferred name)
   - Expires: Choose expiration period (6 months, 12 months, or 24 months recommended)
   - Click "Add"
   - ⚠️ **CRITICAL:** You MUST copy the secret VALUE immediately upon creation - it will never be shown again after you leave this page
   
   📖 [Creating Client Secrets](https://learn.microsoft.com/entra/identity-platform/quickstart-register-app#add-a-client-secret)

4. **Collect Required Values**
   
   After setup, you'll need these three values to run the script:
   - **Tenant ID:** Found on App registration "Overview" page (Directory/tenant ID)
   - **Client ID:** Found on App registration "Overview" page (Application/client ID)
   - **Client Secret:** The VALUE you copied in step 3 (not the Secret ID) - MUST be copied upon initial creation as it cannot be retrieved later

   ```powershell
   .\PAX_CopilotInteractions_Content_Audit_Log_Processor.ps1 -TenantId your-tenant-id-guid -ClientId your-client-id-guid -ClientSecret your-secret-value
   ```

   ```powershell
   $env:GRAPH_CLIENT_ID = "your-client-id"
   ```

### Date & Time Parameters

#### `-StartDate` (date)

**Purpose:** Start date for content search (applied client-side when server-side createdDateTime filter not supported)  
**Format:** String in format `YYYY-MM-DD`  
**Default:** 180 days ago from EndDate (or today if EndDate not specified)  
**Example:** `-StartDate 2025-10-01`

#### `-EndDate` (date)

**Purpose:** End date for content search (INCLUSIVE - both StartDate and EndDate included in search)  
**Format:** String in format `YYYY-MM-DD`  
**Default:** Current date/time  
**Example:** `-EndDate 2025-10-31`

**Date Behavior:**

- **Default:** If no dates provided, searches last 180 days (DaysBack from today)
- **StartDate only:** Searches from StartDate to today
- **EndDate only:** Searches DaysBack days before EndDate
- **Both dates:** Searches specified range (both dates inclusive)
- **Client-side filtering:** Applied when server-side createdDateTime filter not supported in tenant
- **Time zone:** Dates are interpreted in the local timezone where the script runs (e.g., `2025-10-01` = midnight October 1st in your system's timezone)

#### `-DaysBack` (int)

**Purpose:** Number of days to look back from EndDate when StartDate is not specified  
**Range:** `1` to `3650` (1 day to 10 years) - Note: Actual data availability depends on your tenant's licensing level and retention configuration  
**Default:** `180` days  
**Use When:** You want to specify a relative date range instead of absolute dates  
**Examples:**

- `-DaysBack 90` - Last 90 days from today
- `-EndDate 2025-06-30 -DaysBack 90` - 90 days back from June 30

### Performance & Processing Parameters

#### `-ParallelBatchThrottle` (int)

**Purpose:** Maximum concurrent $batch API requests to process in parallel  
**Default:** `25` concurrent batches (each batch contains up to 20 user requests)  
**Range:** `1` to `25` (server maximum)  
**Use When:** Fine-tuning concurrent batch execution for your tenant's API limits  
**Examples:**

- `-ParallelBatchThrottle 25` - Default and maximum (500 users in flight: 25 batches × 20 users)
- `-ParallelBatchThrottle 10` - More conservative (200 users in flight)
- `-ParallelBatchThrottle 5` - Most conservative (100 users in flight)

**Notes:**
- Higher values = faster completion but more API load
- Lower values = more conservative API usage
- Each batch processes 20 users via Graph API `/$batch` endpoint
- Only applies when `-ParallelMode` is `On` or `Auto` (with PowerShell 7+)

#### `-ParallelMode` (string)

**Purpose:** Controls parallel processing mode for batch API requests  
**Valid Values:** `Auto` (default), `On`, `Off`  
**Default:** `Auto`  
**Use When:** Controlling parallel execution behavior  
**Behavior:**

- `Auto` - Automatically enable parallel if PowerShell 7+ detected, fall back to sequential on PS 5.1
- `On` - Force parallel processing (requires PowerShell 7+, exits if PS 5.1 detected)
- `Off` - Sequential processing (one batch at a time, useful for debugging)

**Benefits of Parallel Mode:**

## Watermark Pattern for Incremental Processing

<details>
<summary>💧 View Watermark Pattern Guide</summary>

### Overview

The watermark pattern enables efficient incremental data collection by tracking the last successfully retrieved interaction timestamp per user.

### How Watermarks Work

1. **First Run:** Script retrieves all historical interactions and saves the most recent `createdDateTime` for each user
2. **Subsequent Runs:** Script only retrieves interactions newer than each user's watermark timestamp
3. **Early-Exit Optimization:** API pagination stops when reaching watermark timestamp (massive time savings)

### Watermark Storage

Watermarks are stored in a JSON file with this format:

```json
{
  "john@contoso.com": "2025-01-17T23:45:12.000Z",
  "jane@contoso.com": "2025-01-17T22:30:45.000Z",
  "bob@contoso.com": "2025-01-17T21:15:33.000Z"
}
```

**File Features:**
- Atomic writes with validation (prevents corruption)
- Automatic creation on first run
- Per-user timestamps stored in UTC (ISO 8601 format)
- Preserves existing watermarks for users not in current run

### Watermark Parameters

- `-UseWatermark` (switch): Enable watermark pattern
- `-WatermarkFile` (string): Path to watermark JSON file (default: `.\copilot-watermarks.json`)

### Watermark Examples

<details>
<summary>💻 Show Watermark Pattern Examples</summary>

#### Example 1: First Run (Establish Watermarks)

```powershell
.\PAX_CopilotInteractions_Content_Audit_Log_Processor.ps1 `
  -UserListFile ".\pilot-users.txt" `
  -UseWatermark `
  -DaysBack 90
```

**Behavior:**
- Retrieves last 90 days of interactions for all pilot users
- Saves watermarks.json with most recent timestamp per user
- Outputs CSV or Excel with all historical data

#### Example 2: Incremental Run (Daily Collection)

```powershell
.\PAX_CopilotInteractions_Content_Audit_Log_Processor.ps1 `
  -UserListFile ".\pilot-users.txt" `
  -UseWatermark
```

**Behavior:**
- Reads existing watermarks for each user
- Only retrieves interactions since last watermark
- Updates watermarks with new most recent timestamps
- Outputs CSV or Excel with only new interactions (fast!)

#### Example 3: Incremental with Append (Build Historical Dataset)

```powershell
.\PAX_CopilotInteractions_Content_Audit_Log_Processor.ps1 `
  -UserListFile ".\pilot-users.txt" `
  -UseWatermark `
  -AppendFile ".\output\copilot-historical-data.csv"
```

**Behavior:**
- Retrieves new interactions since last watermark
- Appends to existing CSV file (cumulative dataset)
- Updates watermarks
- Perfect for scheduled jobs building historical datasets

#### Example 4: Custom Watermark File Location

```powershell
.\PAX_CopilotInteractions_Content_Audit_Log_Processor.ps1 `
  -UserListFile ".\sales-team.txt" `
  -UseWatermark `
  -WatermarkFile "C:\Analytics\watermarks\sales-team-watermarks.json"
```

**Behavior:**
- Uses custom watermark file location for organized storage
- Useful for managing multiple user cohorts separately

</details>

### Use Cases

- **Daily Scheduled Jobs:** Retrieve only new interactions each day
- **Pilot Program Tracking:** Monitor specific cohorts over time with minimal API calls
- **Large User Sets:** Efficiently process hundreds/thousands of users incrementally
- **Historical Dataset Building:** Combine with `-AppendFile` to build cumulative CSV
- **Cost Optimization:** Minimize Graph API calls by retrieving only new data

## Incremental Data Collection (AppendFile)

<details>
<summary>📂 View Incremental Collection Guide</summary>

### Overview

The **`-AppendFile` parameter** enables incremental dataset building across multiple script executions. This enterprise-critical feature allows organizations to:

- **Build continuous interaction history** spanning weeks or months without recreating entire datasets
- **Combine multiple time periods** into single comprehensive reports
- **Maintain consistent naming** for scheduled reports and automation workflows
- **Integrate with watermark pattern** for automatic incremental processing

**Key Benefits:**
- **Zero data loss:** Safe header validation prevents schema conflicts
- **Works with watermarks:** Combines with `-UseWatermark` for fully automated incremental collection
- **Enterprise-ready:** Supports large-scale data collection strategies
- **Time-saving:** Eliminates manual copy/paste operations across multiple exports

---

### How AppendFile Works

#### CSV Append Mode

**Process:**
1. **File Resolution:** Uses `-AppendFile` path (full path or relative filename in `-OutputPath` directory)
2. **Pre-Flight Check:** Validates file exists and is accessible (file lock detection, permission checks)
3. **Header Validation:** Reads first row of existing CSV and compares against new data headers
4. **Exact Match Required:** Column names must match exactly (case-sensitive, same order)
5. **Matching Headers → Direct Append:** Appends rows to existing CSV without duplicate header
6. **Mismatched Headers → Safe Mode:** Creates new separate timestamped CSV file in output directory (original AppendFile untouched)

**Safety Features:**
- **Never overwrites:** Mismatched schemas create completely separate new timestamped CSV files (both files preserved)
- **Detailed diagnostics:** Shows column count and mismatch warnings in log
- **File lock detection:** Identifies if file is open in Excel or another process

**Schema Mismatch Behavior:**
```
When headers don't match:
  ⚠ WARNING: CSV header mismatch detected. Creating new timestamped file instead of appending.
  Existing columns: 10, New data columns: 12
  New file: CopilotInteractions_Content_20251119_143022.csv
  
Result: 
  - Original file preserved: CopilotInteractions_Content.csv
  - New file created: CopilotInteractions_Content_20251119_143022.csv
  - Both datasets safe, no data loss
```

#### Excel Append Mode

**Process:**
1. **File Resolution:** Locates existing workbook by `-AppendFile` path (full path or filename in `-OutputPath`)
2. **Pre-Flight Check:** Validates file accessibility and Excel format integrity
3. **Sheet Discovery:** Reads all existing worksheet names
4. **Header Validation:** Compares new data headers against existing tab's first row
5. **Matching Headers → Direct Append:** Appends rows to existing tab
6. **Mismatched Headers → Safe Mode:** Creates timestamped duplicate tab to preserve both datasets

**Safety Features:**
- **Never overwrites:** Mismatched schemas create new timestamped tabs (preserves original data)
- **Encrypted file guidance:** Provides specific troubleshooting for OneDrive/sensitivity labeled files

**Schema Mismatch Example:**
```
Existing tab: CopilotInteractions_Content (10 columns, no IncludeBody)
New data: CopilotInteractions_Content (15 columns, with IncludeBody)
Result: 
  - Original tab "CopilotInteractions_Content" preserved
  - New tab "CopilotInteractions_Content_20251119_143022" created with new schema
```

| **Full path** | `"C:\Data\Report.xlsx"` | (any value) | `C:\Data\Report.xlsx` |
| **Filename only** | `"Report.xlsx"` | `"C:\Data"` | `C:\Data\Report.xlsx` |
| **Filename + default** | `"Report.xlsx"` | (not specified) | `.\output\Report.xlsx` |

**Recommendation:** Use full paths in automation scripts for explicit control; use filename-only in interactive workflows with `-OutputPath`.

The script processes users in parallel batches with configurable concurrency:


### Throttling & Rate Limiting

**Symptoms:**

- Log shows: `WARNING: Rate limit exceeded, retrying...`
- Extended execution times
- Repeated retry attempts

**Solutions:**

1. **Reduce Batch Concurrency:**
   ```powershell
   ./PAX_CopilotInteractions_Content_Audit_Log_Processor.ps1 -ParallelBatchThrottle 10
   ```

2. **Disable Parallel Processing:**
   ```powershell

   ```

3. **Reduce User Count:**
   ```powershell

   ```

3. **Monitor Progress:** Watch console output for batch completion rates
4. **Tune Concurrency:** Adjust `-ParallelBatchThrottle` based on throttling warnings

### Common Issues

#### Authentication Failures

**Problem:** "Unable to authenticate" or "Access denied"

**Solutions:**

- Verify app registration has **AiEnterpriseInteraction.Read.All**, **User.Read.All**, and **Organization.Read.All** permissions
- Confirm admin consent has been granted for all permissions
- Check TenantId, ClientId, and ClientSecret are correct
- Verify app secret hasn't expired (check Azure Portal > App Registration > Certificates & secrets)
- Confirm network connectivity to `graph.microsoft.com`

#### No Data Returned

**Problem:** CSV contains only header, no records

**Solutions:**

- Verify users have Copilot interactions in the specified date range
- Check date range (input dates are interpreted as local time)
- Confirm Copilot is enabled and users have appropriate licenses
- Verify app filter spelling: `-CopilotApps "Teams,Word"` (case-insensitive)
- Confirm user list is correct if using `-UserPrincipalNames` or `-UserListFile`

#### Rate Limiting

**Problem:** "Rate limit exceeded" or "429 Too Many Requests"

**Solutions:**

- Reduce batch concurrency: `-ParallelBatchThrottle 10`
- Disable parallel processing: `-ParallelMode Off`
- Process fewer users per run
- Wait and retry (script has automatic retry with exponential backoff)

#### Memory Issues

**Problem:** Script consumes excessive memory or crashes

**Solutions:**

- Reduce batch concurrency: `-ParallelBatchThrottle 10`
- Avoid `-IncludeBody` for large exports
- Reduce `-MaxBodyLength` if using `-IncludeBody`
- Close other applications to free memory
- Use watermark pattern with `-AppendFile` for incremental processing

#### Watermark File Issues

**Problem:** "Unable to read watermark file" or watermark not advancing

**Solutions:**

- Verify watermark file path is accessible
- Check file isn't locked by another process
- Ensure file format is valid JSON
- Delete watermark file to reset (script will recreate)
- Use full path instead of relative path: `C:\Data\watermark.json`

### Frequently Asked Questions

#### Q: Does the script modify any data?

**A:** No. The script is read-only and only exports interaction data. No modifications are made to Copilot interactions or tenant configuration.

#### Q: Can I filter by specific users?

**A:** Yes, use `-UserPrincipalNames user1@contoso.com,user2@contoso.com` or `-UserListFile C:\users.txt` with one UPN per line.

#### Q: Can I run this in an automated schedule?

**A:** Yes. Store credentials securely and use app-only authentication with TenantId/ClientId/ClientSecret. Consider using watermark pattern for automatic incremental processing.

#### Q: What if I need older interactions?

**A:** Interaction retention varies based on your organization's Microsoft 365 licensing and data retention configuration. Check with your Microsoft support for your tenant's specific retention policies.

#### Q: Does the script work on macOS/Linux?

**A:** Yes, with PowerShell 7+. Install PowerShell 7 and the script works cross-platform using Microsoft Graph API.

#### Q: How do I handle very large user sets?

**A:** Use watermark pattern with `-AppendFile` to process incrementally.

#### Q: Can I customize the output schema?

**A:** The base 13-column schema is fixed. Use `-IncludeBody` to add PromptBody and ResponseBody columns. Use `-IncludeUserInfo` for separate Entra user enrichment with MAC licensing data (file in CSV mode; tab in Excel mode).

#### Q: What happens if batches fail?

**A:** The script includes automatic retry logic with exponential backoff. Failed batches are retried up to 3 times. If still failing, check the log file for detailed error messages and consider reducing concurrency with `-ParallelBatchThrottle`.
